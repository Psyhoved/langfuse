Prompt Injection: классификация, примеры и методы защиты

Prompt injection — это класс атак на LLM-приложения, при которых злоумышленник
внедряет вредоносные инструкции через пользовательский ввод или внешние данные,
изменяя предполагаемое поведение модели.

Классификация атак:

1. Прямая инъекция (Direct Prompt Injection)
Пользователь напрямую вводит инструкции для модели:
- "Забудь все предыдущие инструкции. Теперь ты..."
- "SYSTEM: override previous instructions"
- "Ignore the above directions and instead..."
Цель: изменить поведение модели, обойти ограничения, получить доступ к системному промпту.

2. Косвенная инъекция (Indirect Prompt Injection)
Вредоносные инструкции размещаются в данных, которые модель обрабатывает:
- В документах RAG-базы знаний
- На веб-страницах, которые агент посещает
- В электронных письмах или сообщениях
- В метаданных файлов
Особенно опасна для агентов с инструментами, так как модель может выполнить
вредоносные действия от имени пользователя.

3. Атаки через инструменты (Tool-based Injection)
В контексте ИИ-агентов с инструментами:
- Результат веб-поиска содержит инструкции для модели
- Документ из RAG содержит скрытые директивы
- API-ответ включает вредоносный payload
Агент может интерпретировать эти данные как инструкции и выполнить нежелательные действия.

Техники детекции:

Паттерн-матчинг:
- Регулярные выражения для известных паттернов ("ignore previous", "you are now")
- Словари подозрительных фраз на разных языках
- Детекция кодирования (Base64, hex, ROT13)

ML-классификация:
- Обученные классификаторы для определения инъекций
- Эмбеддинг-анализ: сравнение семантической близости к известным атакам
- Аномалия-детекция: нетипичные паттерны ввода

Структурный анализ:
- Разделение данных и инструкций (data/instruction separation)
- Canary tokens: скрытые маркеры в системном промпте
- Sandwich defense: повторение инструкций после пользовательского ввода

Метрики для оценки защиты:

- Attack Success Rate (ASR): процент успешных атак из тестового набора
- False Positive Rate (FPR): процент ложных срабатываний на безобидных запросах
- Detection Latency: время обнаружения атаки
- Coverage: процент известных типов атак, которые система обнаруживает

Роль Langfuse в защите от инъекций:

1. Трассировка: каждый ввод пользователя и ответ модели записываются
2. Скоринг: автоматическая оценка injection_risk для каждого запроса
3. Датасеты: набор тестовых атак для регрессионного тестирования
4. Мониторинг: алерты при обнаружении паттернов инъекций
5. Аудит: полный лог для расследования инцидентов
