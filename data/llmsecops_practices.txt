LLMSecOps: практики безопасности для операционализации LLM

LLMSecOps — это дисциплина, объединяющая практики безопасности (Security),
операционного управления (Operations) и разработки (Development) для LLM-приложений.
По аналогии с DevSecOps, но специфичная для систем на основе языковых моделей.

Принципы LLMSecOps:

1. Security by Design
Безопасность закладывается на этапе проектирования, а не добавляется постфактум.
- Threat modeling перед разработкой
- Принцип наименьших привилегий для инструментов агента
- Разделение контуров обработки данных разной чувствительности

2. Continuous Monitoring
Постоянный мониторинг поведения LLM в продакшене.
- Автоматическая оценка каждого ответа
- Отслеживание аномалий в стоимости и латентности
- Алерты при обнаружении подозрительных паттернов

3. Defense in Depth
Многоуровневая защита:
- Input validation → LLM guardrails → Output validation → Post-processing
- Каждый уровень ловит то, что пропустил предыдущий

4. Auditability
Полная прослеживаемость действий:
- Каждый запрос и ответ записывается с метаданными
- User ID и Session ID для идентификации источника
- Immutable audit log

Практические паттерны:

PII Filtering (фильтрация персональных данных):
Перед отправкой данных в LLM или систему наблюдаемости:
- Маскирование email-адресов: user@example.com → [EMAIL_REDACTED]
- Маскирование номеров карт: 4111-1111-1111-1111 → [CARD_REDACTED]
- Маскирование телефонов: +7-999-123-45-67 → [PHONE_REDACTED]
- Маскирование ИНН, СНИЛС и других идентификаторов

Cost Monitoring (мониторинг стоимости):
- Установка бюджетных лимитов на пользователя/сессию
- Алерты при превышении порогов (>2σ от среднего)
- Детекция "runaway loops" (агент зацикливается и тратит токены)
- Ежедневные отчёты по стоимости с разбивкой по моделям

Prompt Security:
- Версионирование промптов через Langfuse Prompt Management
- Code review для изменений системных промптов
- Canary tokens для детекции утечки промптов
- Разделение системных и пользовательских данных в промпте

Safety Evaluation:
- Создание датасетов с атаками (prompt injection, jailbreak)
- Регулярный прогон экспериментов при обновлении промптов или моделей
- Метрики: Attack Success Rate, False Positive Rate
- Regression testing: новая версия не должна быть хуже предыдущей

Инструменты LLMSecOps:

Наблюдаемость: Langfuse (self-hosted), LangSmith, Phoenix
Guardrails: NeMo Guardrails, Guardrails AI, LLM Guard
Тестирование: Giskard, Promptfoo, HuggingFace Evaluate
Мониторинг: Grafana + Langfuse metrics, PagerDuty алерты

Workflow LLMSecOps с Langfuse:

1. Разработка: создание промптов в Langfuse Prompt Management
2. Тестирование: датасеты с атаками, эксперименты, скоринг
3. Деплой: промпт переводится в статус "production"
4. Мониторинг: автоматические скоры, мониторинг стоимости
5. Инцидент: трассировка для расследования, обновление датасета
6. Улучшение: новая версия промпта → возврат к шагу 2
